Buddy — iPhone Shortcut & Quick Capture Guide

Single endpoint, one API key, AI by default, flags to skip the AI. Updated 2026-05-13.

1. The new picture

Before: your Todo Shortcut posted text to /functions/v1/quick-note. That endpoint only saved notes with simple -flag sorting. No AI, no tasks, no reminders, no confirmation.

After: one Shortcut posts text to /functions/v1/assistant — the same backend the in-app chat uses. It supports:

  • Plain text → AI classifies and routes (note, task, reminder, mood, journal, check-in, etc.).
  • Slash commands (/task, /remind …) → skip the AI, zero cost.
  • Dash flags (-task, -remind …) → same as slash commands, easier to type on mobile.
  • JSON response with action_taken → show as an iPhone notification to confirm what fired.

2. Authentication (no change)

Keep using the API key from Account → Quick Capture API. The assistant endpoint accepts the exact same key (it looks up the same quick_note_api_key setting). You don't need to regenerate.

4. The flag system (refined)

Three tiers run before the AI; the first one to match wins. AI is the fallback.

  1. Slash command — /task fix bike by friday
  2. Dash flag — -task fix bike by friday (translates to /task …)
  3. Regex rule — natural-language patterns (e.g. "herinner me om 14:00…")
  4. AI — anything else (default route)

Reminders & extras (the "-remind needs a time" case)

When a flag needs an extra (time, value, score), include it in the same line. Examples that work without AI:

-remind 14:00 call dentist
-remind 20:30 take supplements
-remind in 2 uur opbellen
-checkin sleep 7 energy 3 mood 4
-mood 4 long walk this morning
-study Linear algebra 90m
-goal Run 5km by 2026-09-01

If the backend can't parse the extra (e.g. -remind buy milk with no time), the response comes back as:

{
  "success": false,
  "action_taken": "Reminder needs a time. Try \"/remind 14:00 call dentist\" or \"in 2 uur opbellen\".",
  "data": { "needs_param": "time", "example": "/remind 14:00 call dentist" }
}

Your Shortcut already shows action_taken as a notification, so you'll see this on your lock screen and can try again. If you want, add a follow-up step for reminders specifically:

Two-step reminder Shortcut.
1. Ask for Input — "What should I remind you of?"
2. Ask for Input — "When?" (e.g. 14:00 or in 2 uur)
3. Text action — combine into -remind {When} {What}
4. Get Contents of URL — same as above, with that text as input.
This is the closest you can get to "iPhone prompts for extras". Shortcuts doesn't support reading a response and asking another question in one chain without you wiring it up explicitly.

7. Troubleshooting

SymptomCheck
401 Authentication failedAPI key wrong or not saved. Regenerate from Account.
400 input is requiredBody JSON missing input. Verify Step 2 has Request Body set to JSON, not Form.
Notification shows raw JSONStep 3 missing — add "Get Dictionary Value" for action_taken before "Show Notification".
Reminders never fireYou need a working push subscription. Open the web app once, allow notifications, confirm in Account.
Free text routed to wrong toolAdd a flag prefix to force routing — flags always beat the AI.

8. What changed in this rehaul

  • Endpoint switch: Shortcut now points at /functions/v1/assistant (full backend) instead of /functions/v1/quick-note (notes only).
  • Flag system expanded: 14 new flag aliases (EN + NL) in command-parser.ts — every common slash command has a dash twin.
  • Better reminder errors: when a time is missing the response includes needs_param: "time" with a usable example.
  • Account page docs: updated to show new endpoint, body shape, and the full flag cheat sheet.
